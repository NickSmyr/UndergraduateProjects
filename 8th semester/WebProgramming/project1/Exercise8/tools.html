Εργαλεία

Πως δουλεύει πρώτα η Μηχανική Μάθηση;

Στο πρόβλημα της υπολογιστικής όρασης προσπαθούσαμε να φτιάξουμε ένα πρόγραμμα που να αναγνωρίζει πράγματα σε εικόνες. Επειδή όμως δεν υπάρχει κάποιος μαθηματικός τύπος για την παρουσία του τύπου δεν είναι εύκολο να φτιαχτεί ενα πρόγραμμα με κανόνες για το αν υπάρχει ένας σκύλος σε κάποια δεδομένα εισόδου.

Ωστόσο τι θα γινόταν αν μπορούσαμε να δείξουμε στο πρόγραμμα πολλές φωτογραφίες σκύλων ώστε να μάθει μόνο του τι μορφή έχει ο σκύλος; Αυτή είναι η ιδέα πίσω απο την μηχανική μάθηση. Αφού έχει δεί τόσες φωτογραφίες σκύλων ένας αλγόριθμος μηχανικής μάθησης φτιάχνει τον δικό του μαθηματικό τύπο που αντιστοιχεί μια εικόνα σε μια ετικέτα. Η ίδες ιδέα εφαρμόζεται και στις άλλες εφαρμογές του Deep Learning όπως την επεξεργασία φυσικής γλώσσας.

Πως μαθαίνεται ο μαθηματικός τύπος

Συνήθως, οι μαθηματικοί τύποι ενός μοντέλου μηχανικής μάθησης είναι περιορισμένα σε μια οικογένοια συναρτήσεων για παράδειγμα οι a * x + b. Σκόπος τότε του μοντέλου είναι να βρεί αυτές τις μεταβλητές a, b ωστε οι έξοδοί του να ταιριάζουν όσο περισσότερο γίνεται σε αυτά που θέλουμε να είναι (π.χ στην υπολογιστική όραση θέλουμε η είσοδος να είναι μια εικόνα και η έξοδος ένας αριθμός-αναγνωριστικό του αντικειμένου που περιέχεται).

Το πόσο καλά ταιρίαζει στα δεδομένα ο αλγόριθμος μπορούμε να το περιγράψουμε μέ ένα αριθμό, το λεγόμενο loss function. Υπάρχουν πολλοί τρόποι να το όρίσουμε αλλα σε κάθε περίπωση γνωρίζουμε πως αν το loss φτάσει να είναι 0, τότε ο αλγόριθμος βγάζει ακριβώς τις εξόδους που θέλουμε, είναι τέλειος δηλαδή. Άρα το μοντέλο το μόνο που χρειάζεται να κάνει είναι να αλλάξει τις παραμέτρους του ώστε να μειωθεί το loss.

Όμως ο μοντέλο θέλει κάποια στρατηγική για να αλλάζει τις παραμέτρους του. Δεν μπορεί απλα στην τύχη να δοκιμάζει διάφορα νούμερα γιατί δεν θα τελειώσει ποτέ. Εδώ είναι που χρειάζεται να υπολογίσουμε τις παραγώγους. Ο μαθηματικός ορισμός της παραγώγου μιας συνάρτησης λέει πως αν κουνήσουμε τις παραμέτρους προς την κατεύθυνση της τότε θα μεγαλώσει η τιμή της συνάρτησης. Άρα αρκεί να κουνηθούμε προς την αντίθετη κατεύθυνση για να μειώσουμε μια συνάρτηση, που είναι αυτό που θέλουμε για το loss (που είναι συνάρτηση των παραμέτρων του μοντέλου). Η παραπάνω διαδικασία καλείται εκπαίδευση του μοντέλου.

Συνοψίζοντας, για να βρούμε το τέλειο μοντέλο ακολουθούμε την παράγωγο του loss ως προς τις παραμέτρους του μοντέλου έως ότου η κατεύθυνση του να γίνει 0, δηλαδή εως ότου να μην μπορούμε να βετλτιώσουμε το μοντέλο άλλο.

Deep Learning

Ώς τώρα οι ιδέες που έχουμε δεί είναι ιδέες της μηχανικής μάθησης. Όμως η διαφορά της μηχανικής μάθησης με το Deep Learning βασίζεται στις συναρτήσεις που χρησιμοποιούμε στο Deep Learning. Το Deep Learning όπως προαναφέρθηκε βασίζεται στην βιολογία και στους νευρώνες. Με απλά λόγια, θέλει να προσομοιώσει έναν εγκέφαλο. Ο εγκέφαλος λοιπόν έχει εκατομμύρια με δισεκατομμύρια νευρώνες και το Deep Learning αυτό προσπαθέι να κάνει. Συγκεκριμένα, για ένα συγκεκριμένο διάνυσμα x εφαρμόζεί γραμμικές συναρτήσεις όπως ο πολλαπλασιασμός πινάκων και μη γραμμικές συναρτήσεις ενεργοποίησης επανειλημμένα. Το αποτέλεσμα είναι το μοντελο να μπορεί να έχει εκατομμύρια η δισεκατομμύρια παραμέτρους. Αυτά τα μοντέλα τα οποία είναι πολύ βαθίά έχουν την δυνατότητα να ταιριάξουν πολλές περίπλοκες συναρτήσεις. Μια ιδανική συνάρτηση για την ευρεση αντικειμένων σε εικόνες είναι τώρα κατα πάσα πιθανότητα μια ανάθεση τιμών για αυτές τις παραμέτρους.

Τα Εργαλεία του Deep Learning

Τα εργαλεία που διευκολύνουν το Deep Learning είναι τα frameworks που παρέχουν τα κατάλληλα API για να γραφτεί κώδικας που να περιγράφει τα μοντέλα που περιγράψαμε παραπάνω, καθώς και εξειδικευμένο hardware που να επιτρέπει την βελτιστοποίηση τέτοιων μοντέλων.

Τα πιό διάσημα framework είναι το PyTorch της Facebook και το TensorFlow της Google. Και τα δύο κατα κόρων παρέχουν την δυνατότητα για αυτόματο υπολογισμό παραγώγων. Όπως είδαμε αναγκαίο για την εκπαίδευση των μοντέλων είναι ο υπολογισμός της παραγώγου του loss function, αλλα ο αναλυτικός υπολογισμός παραγώγων με το χέρι όπως γνωρίζουμε απο τα μαθηματικά λυκείου είναι δύσκολος, πόσο μάλλων όταν η συνάρτηση είναι τόσο μεγάλη. Για αυτό τον λόγο αυτά τα frameworks αναλαμβάνουν τον υπολογισμό τους· Εμείς ορίζουμε την συνάρτηση και αυτά υπολογίζουν τις παραγώγους αυτόματα.

Τέλος το hardware που χρησιμοποιείται είναι οι κάρτες γραφικών, μόνο που οι δυνατότηες που έχουν για την επεξεργασία 3D γραφικών εφαρμόζονται για τον υπολογισμό πράξεων σε τανυστές (tensors), γενίκευση του πίνακα απο την γραμμική άλγεβρα. Ο λόγος που οι κάρτες γραφικών είναι τόσο ελκυστικές για την κοινότητα του Deep Learning είναι πως οι πράξεις για τους τανυστές περιέχουν πολλή παραλληλία, δηλαδή μπορούν να εκτελεστούν πιό εύκολα αν έχουμε πολλές μονάδες επεξεργασίας (CPU cores), και οι κάρτες γραφικών περιέχουν εκατοντάδες τέτοιες μονάδες.
